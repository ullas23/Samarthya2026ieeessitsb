=========================================================
TECHNICAL REPORT: Samarthya 2026 - Forge of Innovation
=========================================================

1. PROJECT OVERVIEW
---------------------------------------------------------
Samarthya 2026 is the flagship technical festival of the IEEE SSIT Student Branch. The web application serves as the central hub for the festival, featuring a cinematic, Norse mythology-themed interactive experience. The project is designed to be highly immersive, visually striking, and fully responsive across all device sizes. It focuses heavily on advanced front-end web technologies to deliver an impact-driven narrative.

2. TECHNOLOGY STACK
---------------------------------------------------------
The project avoids heavy frameworks (like React or Vue) in favor of lightweight, highly performant Vanilla HTML, CSS, and JavaScript. 

- Core Structure: HTML5
- Styling: Custom Vanilla CSS3 (CSS Variables for theming, Flexbox/Grid for layout, Clamp() for responsiveness)
- Interactivity & Logic: Vanilla JavaScript (ES6+)
- 3D Graphics & Rendering: Three.js (r128)
- Animations & Scrolling: GSAP (GreenSock Animation Platform 3.12.2) and ScrollTrigger plugin
- Typography: Google Fonts (Cinzel, Cinzel Decorative, Rajdhani, Share Tech Mono)

3. DIRECTORY STRUCTURE
---------------------------------------------------------
The application resides primarily within the 'S_B' directory with a clean asset-separated structure:
- S_B/
  - index.html        (Main Single-Page Application entry point)
  - Css/              (Stylesheet directory)
      - loader.css    (Styles for initial loading animation)
      - mobile-fix.css(Mobile responsiveness patches)
  - Js/               (JavaScript logic directory)
      - loader.js     (Loading state and cinematic intro logic)
      - mobile-fix.js (Mobile-specific interactivity adjustments)
  - Assets/           (Images, 3D models, textures, background art like Yggdrasil)
  - Logos/            (Dynamic loading directory for society logos)
  - Past/             (Past events or archival assets)
  - Rules/            (Event rulebooks, manuscript-style content)

4. ARCHITECTURE & DESIGN SYSTEM
---------------------------------------------------------
The design is heavily rooted in a dark, cinematic Norse theme characterized by glowing runes, deep blues (midnight/navy), and metallic gold accents.

- Theming & Variables: A global `:root` system establishes a strict color palette (e.g., `--midnight`, `--navy`, `--gold`, `--cyan`, `--rune-glow`).
- Typography Hierarchy: 
  - 'Cinzel Decorative' for large, impactful titles.
  - 'Share Tech Mono' for technical "eyebrow" text and dates.
  - 'Rajdhani' for readable, modern body text.
- Custom Cursor: A trailing dot cursor with hover-state expansion using JavaScript coordination, replacing the default browser cursor to enhance immersion.
- Performant Responsiveness: Heavy usage of CSS `clamp()` functions allows fluid scaling of typography, padding, and layout bounds without excessive media queries.

5. KEY IMPLEMENTATIONS
---------------------------------------------------------
5.1. Initial Loader & Cinematic Reveal
The loader consists of a Norse mythological sequence (e.g., Hammer drop, Yggdrasil thematic elements). `loader.js` orchestrates a loading state that prevents access until visual assets are buffered, followed by a time-synchronized destruction of the loader overlay, revealing the Hero section behind it.

5.2. Hero Section 3D Canvas
The hero section integrates a `<canvas id="hero-canvas">` driven by Three.js. This allows for dynamic particles, volumetric effects, or object rendering (depending on specific scene logic) to create a sense of depth behind the main typography.

5.3. GSAP Scroll Animations
Instead of static scrolling, GSAP ScrollTrigger defines waypoints across sections (like "Welcome Norse", "About", "Societies") to fade in, scale up, or translate elements as they enter the viewport, giving the site a dynamic flow.

5.4. Dynamic Grid Ecosystem
The societies display and event sections utilize CSS Grid with auto-fit architectures (`grid-template-columns: repeat(auto-fit, minmax(...))`) to inherently handle various screen sizes. Hover states trigger complex CSS animations (like `runeRotate` and inset box-shadow glows).

6. CONCLUSION
---------------------------------------------------------
The modular Vanilla stack ensures high frame-rate animations and rapid loading despite heavy visual fidelity. By carefully balancing Three.js 3D rendering with GSAP DOM manipulation, Samarthya 2026 establishes a premium, engaging aesthetic suitable for a premier technical festival.
